Nix Configuration Deep Review (kuraokami)
=======================================

Scope & methodology
- Reviewed all tracked Nix modules, scripts, CSS, Lua, and YAML in this repo.
- Focused on correctness, security, maintainability, performance, and consistency.
- No runtime validation or rebuilds performed in this review.

Executive summary (prioritized)
1) CRITICAL: Plaintext secrets and private key present in repo (secrets.nix). Remove, rotate, and git-ignore.
2) HIGH: Potentially invalid/fragile settings in audio limits and kernel params; verify or adjust.
3) HIGH: Unpinned runtime plugin download in Neovim and install script robustness.
4) MEDIUM: Configuration drift/duplication and “sharp edges” in boot/network/Wayland/Sway.
5) LOW: Style/consistency, cleanup, and ergonomics.

CRITICAL findings
1) Plaintext secrets in repo (secrets.nix)
   - File contains NAS password, user hash, and an SSH private key.
   - Risk: immediate credential compromise if repo is ever shared/backed up publicly.
   - Action:
     - Delete secrets.nix or move to a non-tracked location.
     - Rotate the SSH key and any exposed passwords.
     - Add secrets.nix to .gitignore if you want a local template.

HIGH findings
2) systemd user limits for PipeWire/WirePlumber may be invalid
   - Files: system/hardware/audio.nix
   - LimitNICE set to 40. Linux nice range is typically -20..19; values outside may be ignored or fail.
   - Action: set LimitNICE to 19 (or remove) and validate via `systemctl --user show`.

3) Kernel params include high-risk flags without guardrails
   - Files: system/core/boot.nix
   - amdgpu.ppfeaturemask=0xffffffff enables all PP features (overclock/undervolt); risky for stability.
   - nmi_watchdog=0 and nowatchdog reduce system diagnostic safety.
   - tsc=reliable can be incorrect on some systems.
   - Action: confirm each flag is needed for this exact hardware; remove non-essential flags.

4) Neovim bootstrap downloads code at runtime
   - File: home/programs/neovim/init.lua
   - Pulls lazy.nvim from GitHub on first run (non-reproducible, network dependency, integrity risk).
   - Action: manage plugins with Nix (e.g., home-manager or nixvim) or vendor/lock plugin source.

5) Install script assumes flake username parsing and uses networked disko run
   - File: install.sh
   - `grep 'username = '` is brittle; changes in flake format break it.
   - `nix run github:nix-community/disko` pulls from network, no pin.
   - Action: pin disko (flake input) or lock revision in script; parse username more robustly or accept prompt.

MEDIUM findings
6) allowUnfreePredicate may be too strict for current package set
   - File: system/core/packages.nix
   - Predicate only allows a few names (steam, claude-code, unrar). If any installed package is unfree (e.g., onlyoffice-desktopeditors, mullvad-vpn depending on nixpkgs), builds will fail unpredictably.
   - Action: audit `home.*` packages and expand allowUnfreePredicate to exact unfree names in use.

7) Multiple modules set portal config; potential conflicts
   - Files: system/desktop/xdg.nix, system/desktop/sway.nix
   - Both set xdg.portal; ensure desired portal set is consistent (wlr vs generic).
   - Action: consolidate portal config in one module.

8) Storage mount lacks explicit network-online dependency
   - File: system/network/storage.nix
   - CIFS automount typically benefits from `x-systemd.requires=network-online.target` and `x-systemd.after=network-online.target`.
   - Action: add those options for reliability at boot/login.

9) GPU config symlink relies on home path
   - File: system/hardware/gpu.nix
   - tmpfiles rule links `/etc/lact/config.yaml` to `/home/${username}/nix-config/...`.
   - If /home is on a separate mount or absent early, the link may be broken.
   - Action: store config under /etc or /var/lib via `environment.etc` or `age` and point lact to it.

10) services.scx and ananicy correctness depends on kernel/availability
   - File: system/hardware/cpu.nix
   - scx scheduler support is kernel-dependent; failures are silent or cause boot issues.
   - Action: ensure kernel supports scx_lavd; set fallback scheduler or gate on kernel version.

11) zramSwap memoryPercent = 50 is aggressive for RAM-constrained systems
   - File: configuration.nix
   - Could lead to heavy compression/latency if physical memory is modest.
   - Action: tune based on RAM size; consider 25–33% or use heuristic.

12) Waybar module styles reference unused IDs
   - File: home/desktop/waybar/style.css
   - Contains #pulseaudio rules but wireplumber is used; remove or align for clarity.

LOW findings / cleanup
13) Duplicate or redundant settings
   - Fonts: defined in system/desktop/fonts.nix and home/home.nix; ensure they align.
   - Sway: bars = [] plus waybar launched; ok, but consider a brief comment explaining no sway bar.
   - `services.xserver.displayManager.lightdm.enable = false` is redundant if xserver itself not used.

14) Small style/consistency improvements
   - Use consistent spacing and comment style across modules.
   - Consider naming conventions for custom scripts (prefix with `waybar-` or similar).

15) Zsh alias `fix-nvim` uses sudo on user files
   - File: home/shell/zsh/zsh.nix
   - Risk: permissions and ownership issues.
   - Action: remove sudo or limit to necessary path with ownership checks.

16) `home/dev/packages.nix` includes `python315`
   - Likely a typo (python311/312/313). If nixpkgs lacks python315 this will fail.
   - Action: verify and correct to the desired interpreter.

Security & privacy notes
- Root password is disabled; good, but ensure wheel user password is set via age secret.
- networkmanager connectivity disabled and geoclue off; aligns with privacy goals.
- Consider enabling `nix.settings.trusted-users = [ "root" "@wheel" ]` if you need non-root Nix usage.

Performance / latency notes
- PipeWire low-latency settings are aggressive; verify with `pw-top` or audio crackle tests.
- `kernelPackages = pkgs.linuxPackages_zen` is tuned for latency; good for desktop, but check module compatibility.
- `WLR_DRM_NO_ATOMIC=1` can reduce performance; only use if needed for stability.

Maintainability / reproducibility
- Replace runtime GitHub downloads (Neovim) with Nix-managed plugins.
- Avoid `nix run github:...` in installer; pin disko in flake inputs.
- Consider a small `README` section listing host-specific assumptions (GPU model, monitors, NAS IP).

Suggested hardening (optional)
- Consider enabling `boot.kernel.sysctl."kernel.kptr_restrict" = 2;` if not already covered.
- Review `protectKernelImage = true` (ok) and consider `security.apparmor.enable = true` if desired.

Notes on “perfect” config
- Perfection depends on hardware, threat model, and usage. The items above are the highest leverage areas.
